🛠 IQC 대시보드 기술문서 Technical Document

← 대시보드로

내부정도관리 종합검토·평가(LM-B-05)를 웹에서 수행하고 전자서명·감사추적·PDF 리포트를 제공하는 Flask 기반 단일 배포형 웹 애플리케이션입니다. 외부 CDN·인터넷 없이 폐쇄망에서 구동되며, 동일 코드가 로컬(단독/인트라넷)과 클라우드(Railway)에서 모두 동작합니다.

1. 기술 스택
언어/런타임Python 3.10+ (배포 3.12)
웹Flask · Flask-SQLAlchemy · Flask-Login
ORM/DBSQLAlchemy 2.0 · SQLite(단일 파일)
프런트엔드Bootstrap 5 · Chart.js (로컬 번들)
리포트ReportLab + NanumGothic(OFL) 임베드 PDF
운영/기타waitress · Pillow · openpyxl · xlrd
2. 시스템 아키텍처

애플리케이션 팩토리(create_app) + 블루프린트 계층형 설계

  • Presentation: Jinja2 + Bootstrap/Chart.js
  • Routing: 블루프린트 11개(auth·dashboard·master·qc·review·capa·audit·account·phrase·settings·admin)
  • Service: 통계·판정·권한·무결성 해시(services.py), Westgard 엔진(westgard.py, 순수 파이썬)
  • Data: ORM 모델(models.py), LIS 파서(importers.py)
  • Report/Seed: pdf_report.py, seed.py+seed_data.py(내장 데모)

4. 설치 · 실행 · 배포

로컬(개발)

pip install -r requirements.txt
flask --app run.py seed   # DB 초기화 + 데모 시드
python run.py             # http://127.0.0.1:5000

인트라넷(운영): python serve.py 또는 서버시작.bat (waitress, 0.0.0.0:5000)

클라우드(Railway): Nixpacks 자동 빌드 → python serve.py. PORT 자동 인식·데이터 비면 자동 시드(내장 데이터로 파일 의존 없음).

6. 핵심 알고리즘 및 설계

6.1 Westgard 멀티룰 — 순수 파이썬 엔진, z-score 시계열 순차 판정. 기본 1₂ₛ·1₃ₛ·2₂ₛ·R₄ₛ·4₁ₛ (10ₓ·8ₓ·7T 보유·미적용).

6.2 이원 평균(CLSI C24) — L-J 중심선은 확립평균(oper_mean), 배정값 편차는 (Mean)%Diff로 별도 평가. 정밀성은 LIS 원본 보존, 위반 건수만 일별 재판정 집계.

6.3 적합/확인 판정

적합 = (당월 CV ≤ 그룹기준) AND (|(Mean)%Diff| ≤ 기준)
        AND (|(CV)Diff| ≤ 기준) AND (일별 reject == 0)
기본 임계값: CV 5% / 저농도 10% / %Diff 3% / CVDiff 3%

6.4 전자서명 무결성 — 검토대상 QC 데이터셋을 직렬화해 SHA-256 해시(자유서술 의견 제외). 서명 시 스냅샷 저장, 조회 시 비교로 변조 탐지. '담당→전문의' 순서 통제 + 비밀번호 재인증.

7. 엔드포인트(API) 명세 (주요)
Method · Path기능
GET /대시보드(장비별 현황)
GET,POST /login · GET /logout인증
GET /master/ · POST /master/importLIS 파일 업로드
GET /qc/chart/<id> · GET /qc/api/series/<id>L-J 차트 화면 / 데이터(JSON)
GET /review/<inst>/<ym>정밀성 종합검토 화면
POST /review/<id>/comment · /sign검토의견 저장 · 전자서명(재인증)
GET /review/<id>/pdfLM-B-05 PDF 생성
GET /capa/ · POST /capa/<id>/resolve시정조치
GET /audit/ · /settings/… · /admin/…감사추적 · 보고서설정 · 관리자
GET /guide · GET /tech사용법 · 기술문서(공개)

※ /qc/api/series/<id>는 JSON(analyte·level·points[date,value,z,status,rules])을 반환해 Chart.js가 L-J 차트를 렌더링. 그 외는 서버 렌더링(Jinja2)이며 인증·역할/장비 권한이 적용됩니다.

8. 보안 · 개인정보
  • Flask-Login + Werkzeug 비밀번호 해시
  • 전자서명: 비밀번호 재인증 + SHA-256 무결성 해시 + 감사추적
  • 권한: 역할 + 장비별 지정 담당자/전문의만 검토·서명
  • 환자 식별정보 미저장(관리물질 QC만). 공개/데모 데이터 익명화
9. 테스트 · 로드맵

멀티룰 단위테스트 13건(pytest) 통과. 통합 스모크(로그인·주요 화면·PDF·권한) 확인.

  • 서명 취소(revoke) 관리, 마우스 서명패드
  • 2단계: EQA·숙련도평가, 장비·시약 관리, 문서 버전관리
  • 3단계: 인증 심사점검표 전반 전자화·다기관 확장
