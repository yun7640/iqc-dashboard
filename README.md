# 내부정도관리(IQC) 웹 대시보드 — 전자서명·감사추적 기반 종합검토 프로그램

> **우수검사실 인증 서류 전자화 및 전자서명 기반 웹 대시보드 개발 — 1단계: 내부정도관리(Internal Quality Control) 프로그램**
> 2026년 진단검사 업무자동화 공모전 (대한임상화학회) 출품작

내부정도관리 종합검토 및 평가(문서번호 **LM-B-05**)를 **종이 출력 없이** 웹 대시보드에서
① 정밀성 종합검토·Levey-Jennings 차트·Westgard 멀티룰 자동판정을 확인하고,
② 정도관리담당·전문의가 검토의견을 작성한 뒤,
③ **비밀번호 재인증 기반 전자서명**으로 승인하며,
④ 모든 행위를 **감사추적(audit trail)** 으로 남기고,
⑤ 인증 제출용 **월간 QC 리포트(PDF)** 를 자동 생성하는 오픈소스 프로그램입니다.

환자 식별정보는 저장·처리하지 않으며, **관리물질(QC material) 기반 정도관리 데이터만** 취급합니다.

> ⚠️ 본 저장소의 시연 데이터·계정명은 모두 **데모용 가명/합성 데이터**입니다. 실제 환자·직원·기관 식별정보를 포함하지 않습니다.

## 2. 기술 스택

- **백엔드** : Python 3.10+, Flask, SQLAlchemy ORM, Flask-Login
- **데이터** : SQLite (단일 파일 DB — 설치 부담 최소화)

- **보안/기록** : Werkzeug 비밀번호 해시, 전자서명·감사추적 테이블, SHA-256 무결성 해시
- **리포트** : ReportLab + 임베드 한글 폰트(NanumGothic, OFL) — 뷰어 폰트 설치 불필요

> 검사실 폐쇄망 환경에서도 그대로 구동됩니다.

## 3. 설치 및 실행

```bash
# 1) 가상환경(권장)
python -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate

# 2) 의존성 설치
pip install -r requirements.txt

# 3) DB 초기화 + 데모 시드 데이터 적재
flask --app run.py seed

# 4) 실행
python run.py
#   → http://127.0.0.1:5000
```

### 검사실 인트라넷 공유 (여러 PC에서 접속)

한 대의 PC(서버 역할)에서 아래처럼 운영용 서버로 실행하면, 같은 네트워크의 다른 PC에서
브라우저로 접속할 수 있습니다.

```bash
pip install -r requirements.txt      # waitress 포함
flask --app run.py seed              # 최초 1회 (DB 초기화)
python serve.py                      # 운영용 실행 (waitress, 0.0.0.0:5000)
```

- 서버 PC에서: `http://127.0.0.1:5000`
- 다른 PC에서: `http://<서버 PC의 IPv4 주소>:5000` (서버 PC에서 `ipconfig` 로 IPv4 확인)
- 서버 PC의 **윈도우 방화벽 인바운드 규칙**에서 해당 포트(기본 5000/TCP)를 허용해야 합니다.
- 포트/주소는 환경변수 `IQC_PORT`, `IQC_HOST` 로 변경할 수 있습니다.
- SQLite 단일 파일 DB는 검사실 규모(소수 동시 사용자)에 적합합니다. 세션 서명키는
  `instance/secret_key.txt` 에 자동 생성·보관되어 재시작에도 로그인 세션이 유지됩니다.
- 서버 PC가 항상 켜져 있도록 하고, 실사용 전 데모 계정 비밀번호를 반드시 교체하세요.

#### 간편 실행 (Windows 배치파일)

- **`서버시작.bat`** — 더블클릭 한 번으로 실행됩니다. DB를 구글드라이브 밖 로컬(`C:\iqc-data`)에
  저장하도록 환경변수를 설정하고, 최초 실행 시 라이브러리 설치·DB 초기화를 자동 수행한 뒤
  운영 서버(`serve.py`)를 시작합니다. (구글드라이브로 코드를 배포·업데이트하되, DB는 로컬에
  두어 동기화 충돌을 방지하는 구성입니다.)
- **`자동시작등록.bat`** — 서버 PC 로그온 시 서버가 자동 실행되도록 작업 스케줄러에 등록합니다.
  (파일 우클릭 → **관리자 권한으로 실행**). 해제: `schtasks /delete /tn "IQC-Dashboard-Server" /f`
- 수동 자동시작(대안): `Win+R` → `shell:startup` → 그 폴더에 `서버시작.bat` 바로가기를 넣으면
  로그인 시 실행됩니다.

### 사용자 관리 · 평가장비별 담당자/전문의 지정 (관리자)

관리자(admin) 계정으로 상단 **관리자** 메뉴에서:
- **사용자 관리**: 계정 추가·역할(정도관리담당/전문의/관리자)·활성화·비밀번호 재설정
- **장비·담당자 지정**: 평가장비 추가 및 **장비별 정도관리담당·전문의 지정(변경 가능)**

지정된 담당자·전문의만 해당 장비의 월별 검토에서 **검토의견 작성·전자서명**이 가능합니다.
지정이 없으면 해당 역할 사용자 누구나 가능하며, 지정 변경은 감사추적에 기록됩니다.

### 항목 특성별 정밀성 평가기준 (그룹)

**보고서설정 → 항목 특성별 평가기준 설정**에서 검사항목의 특성(일반/저농도/면역·정성 등)에 따라
**(Mean)%Diff · (CV)Diff · CV 허용기준을 그룹 단위로 설정**하고 각 검사항목을 그룹에 배정합니다.
배정된 그룹 기준으로 적합/확인 판정이 이루어지며, 월별 검토 화면과 PDF의 '3. 분석 평가기준'에
**특성별로 그룹핑되어** 표기됩니다. 그룹 미배정 항목은 기본 기준(CV ≤ 5% / 저농도 ≤ 10%,
%Diff ≤ 3%, CVDiff ≤ 3%)을 따릅니다.

### 평가장비(기기번호)별 검사항목 배정 · LIS 매칭 점검

**보고서설정 → 평가장비별 검사항목 배정**에서 장비(#1~#4)별로 평가할 검사항목을 선택합니다.
특정 장비에서만 시행하는 항목(#번호 항목)은 해당 장비에만 배정하며, **분석 평가는 장비별로
배정된 검사항목에 대해서만 진행**됩니다. 월별 검토 화면의 '평가항목'은 선택한 장비의 배정 항목과
연동되어 표시되고, **LIS 다운로드 결과와 배정 검사항목이 일치하지 않으면 담당자에게 알림**
(결과 누락 / 미설정 항목)이 표시되어 불일치 원인을 확인하도록 합니다.

### 검토의견 문구 템플릿

상단 **검토문구** 메뉴에서 자주 쓰는 검토의견 문구를 담당자·전문의별로 등록해 두면,
월별 검토 화면의 의견칸에서 **드롭다운으로 선택 → 삽입 → 상황에 맞게 편집 → 저장**할 수 있습니다.
기본 제공 문구(정상·기준충족, 경미 이탈·조치완료, 승인, 조건부 승인 등)가 시드에 포함되어 있습니다.

### 서명 이미지(사인/도장) 등록

상단 **✒ 내 서명** 메뉴에서 각자 사인/도장 이미지(PNG·JPG, 투명 배경 PNG 권장)를 등록하면,
본인이 전자서명할 때 그 이미지가 **대시보드 서명란과 PDF 리포트에 함께 표시**됩니다.
서명 시점의 이미지가 해당 서명 기록에 스냅샷으로 저장되어, 이후 이미지를 바꿔도 과거 서명은 유지됩니다.
(데모 계정에는 예시 서명 이미지가 미리 등록되어 있습니다. 전자서명의 효력은 비밀번호 재인증·무결성
해시·감사추적으로 담보되며, 이미지는 가독성을 위한 시각 표기입니다.)

## 6. Westgard 멀티룰

| 규칙 | 판정 내용 | 오차 유형/조치 |
|---|---|---|
| 1₂ₛ | 1개 값이 ±2SD 초과 | 경고(warning), 추가 규칙 검토 |
| 1₃ₛ | 1개 값이 ±3SD 초과 | 거부, 무작위오차 |
| 2₂ₛ | 연속 2개가 동일 방향 ±2SD 초과 | 거부, 계통오차 |
| R₄ₛ | 인접 두 값의 범위가 4SD 초과 | 거부, 무작위오차 |
| 4₁ₛ | 연속 4개가 동일 방향 ±1SD 초과 | 거부, 계통오차 |
| 10ₓ | 연속 10개가 평균 한쪽에 위치 | **미적용** (검사실 정책) |
| 8ₓ / 7T | 연속 8개 편향 / 연속 7개 추세 (옵션) | 미적용 |

> **검사실 정책상 10ₓ 규칙은 적용하지 않습니다.** 배정평균 대비 소편의(계통적 미세 편차)에
> 과민 반응하여 위양성 위반을 유발하기 때문입니다. 기본 적용 규칙은 1₂ₛ·1₃ₛ·2₂ₛ·R₄ₛ·4₁ₛ 입니다.
> 엔진은 10ₓ·8ₓ·7T 판정 로직을 모두 보유하며 `app/westgard.py` 의 `DEFAULT_RULES` 에서
> 적용 규칙을 조정할 수 있습니다.
>
> L-J 관리한계는 CLSI C24 권고에 따라 **검사실 자체 확립평균(operating mean)** 을 중심선으로,
> 배정 SD를 관리 SD로 사용합니다. 배정값 대비 편차는 정밀성표의 (Mean)%Diff로 별도 평가합니다.

## 7. 평가기준 (LM-B-05 [표3])

- 당월 **CV ≤ 5%** (저농도 물질 ≤ 10%)
- **(Mean) %Diff ≤ 3%** — (해당기간 Mean − 이전기간 Mean) / 이전기간 Mean × 100
- **(CV) Diff ≤ 3%** — 해당기간 CV − 이전기간 CV
